## Описание

Основной идеей моего проекта является предоставление игроку программы, способной автоматически осуществлять настройку и запуск модифицированного Minecraft клиента (Fabric).

Помимо функционала "лаунчера", мной реализованы и некоторые другие вещи, которые в совокупности выстраиваются в полноценную архитектуру, если это так можно назвать.

Важно упомянуть, что я впервые разрабатываю проект такого масштаба. Код может содержать ошибки и не предназначен для продакшн-среды. Проект создаётся в образовательных целях.

Проект является локальным, то есть разрабатывается для меня и моих знакомых, но это не значит, что для остальных вход закрыт. Чтобы принять участие в игре пройдите регистрацию, также заполнив анкету с некоторыми вопросами. После одобрения вашей заявки модератором вы сможете попасть на мой сервер. **НА ДАННЫЙ МОМЕНТ ПРОЕКТ НАХОДИТСЯ НА СТАДИИ НАЧАЛЬНОЙ РАЗРАБОТКИ, СЛЕДОВАТЕЛЬНО ЭТОТ ПУНКТ НЕ АКТУАЛЕН.**

## Разработка

Для начала вам потребуется **yarn** и **docker-compose**. Для работы tauri выполните [инструкции](https://v2.tauri.app/start/prerequisites/) с оф. сайта.

### Установка зависимостей

Выполните установку пакетов с помощью

```sh
yarn install
```

### Настройка .env

В пакетах внутри `apps/`, содержащих в себе `.env.example` создайте `.env` файлы следуя шаблону.

### Запуск контейнеров

В файле `docker-compose.yml` можете изменить имена и пароли БД, в таком случае не забудьте внести эти изменения в конфигурацию prisma! Запускаем контейнеры с Postgres и Redis

```sh
docker-compose up
```

### Настройка Prisma

В пакете `apps/api` необходимо настроить prisma. Сверяем `DATABASE_URL` из `.env`, после генерируем схему

```sh
yarn workspace @repo/api prisma generate
```

и выполняем миграцию

```sh
yarn workspace @repo/api prisma migrate dev
```

### Запуск пакетов

`@repo/api` (бекенд):

```sh
yarn dev:api
```

`@repo/app` (лаунчер):

```sh
yarn dev:app
# или
cd apps/app && yarn dev
```
